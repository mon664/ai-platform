## 11-2. 자동 발주 (Auto-Purchase Order)

### AI 로직
```
1. 매일 아침 안전재고 확인
2. 부족한 원료 자동 감지
3. 기본 발주 수량 계산
   = 안전재고 + (주간 평균 소비 × 2)
4. 거래처별 자동 주문서 생성
5. 사장님 승인만 필요

예시:
- 배추김치 안전재고: 50kg
- 현재 재고: 20kg
- 주간 평균 소비: 15kg
- 발주 수량 = 50 + (15 × 2) = 80kg

사장님에게:
"배추김치 부족 감지.
A푸드에 80kg 주문 대기 중입니다. 
승인해주세요."

사장님: "승인" 
→ 자동으로 주문서 생성 및 발송
```

## 11-3. 품질 관리 (Quality Control)

### AI가 자동으로 수행
```
1. 입고 시 자동 검사
   - 외관 체크 (Claude Vision)
   - 온도 확인
   - 유통기한 확인
   - 포장 손상 여부

2. 생산 중 온도 모니터링
   - 가열 공정: 70°C 이상 30분
   - 냉각 공정: 2시간 이내 20°C 이하
   - 냉장 보관: 4°C ±1°C
   - 냉동 보관: -18°C 이하

3. 배송 전 최종 검사
   - 유통기한 확인
   - 냉장/냉동 온도 확인
   - 포장 상태 확인
   - 수량 확인

4. 이상 발견 시 자동 알림
   ⚠️ "배추김치 유통기한 만료 3일 전
   즉시 판매 또는 폐기 필요"
```

# 14. 문제 해결 가이드

## 14-1. AI 인식 오류 (Vision AI Error)

### 문제 상황
```
송장 사진이 흐릿해서 AI가 잘못 인식
"수량: 100kg" → "수량: 109kg"

해결 방법:
1. 송장 다시 촬영 (명확한 각도)
2. 밝은 곳에서 촬영
3. 테이블 위에 평평하게 펴서 촬영
4. AI 재인식 요청
5. 수동 수정 옵션 제공

개선 방법:
- 송장 촬영 가이드 제시
- 반복 학습으로 정확도 개선
```

---

## 14-2. 재고 불일치 (Inventory Mismatch)

### 문제 상황
```
시스템 재고: 배추김치 20kg
실제 재고: 배추김치 18kg
차이: -2kg

원인 분석:
1. 입고 오류? → 기록 검토
2. 출고 빠뜨림? → 생산기록 검토
3. 폐기? → 폐기 기록 확인
4. 측정 오류? → 정확한 측정 요청

해결 방법:
- 현장 재확인
- 필요시 원인 조사
- 시스템 수정 (감사증적 남김)
- 월말 실사 반영
```

---

## 14-3. API 연동 오류 (API Error)

### 문제 상황
```
ERPNext 연동 실패
"API 호출 실패: Timeout"

원인:
1. 네트워크 불안정
2. ERPNext 서버 다운
3. API 인증 오류
4. 데이터 형식 오류

해결 방법:
1. 네트워크 상태 확인
2. ERPNext 서버 상태 확인
3. API 키 유효성 확인
4. 자동 재시도 (최대 3회)
5. 실패 시 사장님 알림

예방:
- 일 1회 API 정상 작동 확인
- 월 1회 ERPNext 백업 확인
- 비상 연락처 준비
```

---

## 14-4. 계산 오류 (Calculation Error)

### 문제 상황
```
금액 계산 오류
수량 100kg × 단가 15,000원
= 1,500,000원 (정상)

하지만 부가세 계산 오류:
소계 1,500,000원
부가세 150,000원 (10% 계산, 정답은 15%)
합계 1,650,000원 (오류)

검증 로직:
1. 모든 계산 수식 사전 검증
2. 계산 결과 자동 검수
3. 이상 감지 시 자동 알림
4. 수정 후 재계산
```
